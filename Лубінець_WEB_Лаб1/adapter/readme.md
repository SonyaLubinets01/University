# Adapter (Структурний шаблон)

## Опис проблеми
Часто виникає необхідність використати клас зі старим або несумісним
інтерфейсом у новій системі. Зміна існуючого коду може бути неможливою
або небажаною.

---

## Ідея реалізації
Шаблон Adapter створює проміжний клас, який перетворює інтерфейс
одного класу в інтерфейс, очікуваний клієнтським кодом.

---

## UML-схема (спрощена)
Client → Target

↑

Adapter

↑

Adaptee

---

## Де застосовується
- інтеграція сторонніх бібліотек;
- адаптація старого коду;
- API-адаптери;
- міграція між різними системами;
- сумісність різних форматів даних.
